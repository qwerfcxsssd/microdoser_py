Micro Doser

Учебный проект по дисциплине: прикладное программирование / разработка настольных приложений
Платформа: macOS
Язык: Python
GUI: PySide6 (Qt)

Описание проекта

Micro Doser — настольное приложение для ведения приёма лекарств.
Приложение позволяет:
	•	добавлять лекарства вручную;
	•	получать рекомендации и план приёма на основе симптомов;
	•	автоматически формировать напоминания;
	•	просматривать приём лекарств в календаре и ежедневнике.

Проект выполнен в учебных целях и не является медицинским продуктом.

⸻

Основные функции
	•	Главный экран
	•	ежедневник с напоминаниями на выбранную дату;
	•	календарь с подсветкой дней, в которые есть приём лекарств;
	•	быстрые действия: «Подобрать лекарство», «Добавить лекарство».
	•	Подбор лекарства
	•	ввод симптомов пользователем;
	•	обращение к LLM (через OpenRouter API);
	•	получение одного варианта лекарства;
	•	генерация плана приёма и напоминаний.
	•	Добавление лекарства вручную
	•	ввод названия и дозировки;
	•	ввод дополнительной информации (курс, способ приёма и т.д.);
	•	формирование напоминаний на основе введённых данных.
	•	Настройки
	•	резервный ввод API-ключа;
	•	выбор модели;
	•	язык ответа.

⸻

Архитектура проекта

Проект разделён на логические модули:

.
├── main.py                  # точка входа
├── main_window.py           # главное окно и навигация
├── backend/
│   ├── llm_service.py       # работа с LLM (OpenRouter)
│   ├── llm_config.json      # основной конфиг (api key, модель)
│   ├── db.py                # база данных (SQLite)
│   ├── save_llm_result.py   # сохранение результатов
│   └── repositories/        # доступ к данным
├── dialogs/                 # диалоги (подбор / добавление лекарства)
├── screens/                 # экраны (главный, настройки)
├── buttons/                 # UI-компоненты
├── icons/                   # иконки
└── fonts/                   # шрифты

Логика работы с LLM
	•	Основной источник настроек: llm_config.json в корне проекта.
	•	Резервный источник: настройки пользователя (SQLite).
	•	Если файл отсутствует, используется резерв.
	•	Ответ от LLM строго валидируется и приводится к нужной схеме.

⸻

Хранение данных
	•	Используется SQLite.
	•	В базе хранятся:
	•	настройки приложения;
	•	сгенерированные планы;
	•	напоминания для календаря и ежедневника.

Запуск приложения

Готовая сборка

Проект упакован с помощью PyInstaller.
Для запуска достаточно открыть файл приложения с архива на любой архетектуре macOS.

Первый запуск
	•	При использовании функций подбора лекарства требуется API-ключ(изначально там уже стоит
	наш ключ но за его работаспособность не отвечаем).
	•	Ключ можно:
	•	Указать настройках приложения (резервный способ).
